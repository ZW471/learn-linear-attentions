DeltaNet: Delta Rule Linear Attention

Linear Transformers Are Secretly Fast Weight Programmers (Schlag et al., 2021, arXiv:2102.11174)
Gated Delta Networks: Improving Mamba2 with Delta Rule (Yang et al., 2024)

1. The Problem with Standard Linear Attention

Standard linear attention maintains a memory matrix $S_t$ that accumulates key-value associations:

Standard Linear Attention (Recurrent Form) $$S_t = S_{t-1} + v_t \otimes k_t \qquad \text{(write: add new association)}$$ $$y_t = S_t \cdot q_t \qquad \text{(read: retrieve by query)}$$

Where $S_t \in \mathbb{R}^{d_v \times d}$ is a memory matrix (like an associative memory / fast weight matrix), $v_t \otimes k_t$ is the outer product storing "key $k_t$ maps to value $v_t$".

The fatal flaw: This memory only accumulates — it never removes old associations. If key $k$ first maps to value $v_1$ and later should map to $v_2$, the memory contains both $v_1$ and $v_2$, causing interference. Standard attention doesn't have this problem because softmax can shift focus away from old tokens.

Why This Matters: A Concrete Example

Token 1: store("name" → "Alice")
Token 2: store("role" → "engineer")
Token 3: store("name" → "Bob") ← update!
Token 4: retrieve("name") → ???

Standard linear attention: Returns a blend of "Alice" + "Bob" (interference!)
Delta rule: Returns "Bob" (correctly updated)

2. The Delta Rule: Erase Before Write

The delta rule fixes this by computing the difference between the new value and what's currently stored, then writing only the difference:

Delta Rule Update $$\underbrace{S_{t-1} \cdot k_t}_{\text{retrieve old value}} \quad \rightarrow \quad \underbrace{v_t - S_{t-1} \cdot k_t}_{\text{delta (new - old)}} \quad \rightarrow \quad S_t = S_{t-1} + \beta_t \cdot (v_t - S_{t-1} k_t) \otimes k_t$$

Equivalently, this can be written as:

Matrix Form $$S_t = \underbrace{(I - \beta_t \cdot k_t k_t^\top)}_{\text{erase old association}} \cdot S_{t-1} + \underbrace{\beta_t \cdot v_t \otimes k_t}_{\text{write new association}}$$
$S_t$ $\in \mathbb{R}^{d_v \times d}$ — memory matrix
$k_t$ $\in \mathbb{R}^{d}$ — key (normalized)
$v_t$ $\in \mathbb{R}^{d_v}$ — value
$q_t$ $\in \mathbb{R}^{d}$ — query
$\beta_t$ $\in [0, 1]$ — learning rate (input-dependent)
PyTorch-like 
def delta_rule_attention(Q, K, V, beta):
    # Q: (T, d), K: (T, d), V: (T, d_v), beta: (T,)
    # S: (d_v, d) memory matrix
    S = torch.zeros(d_v, d)
    outputs = []

    for t in range(T):
        # Step 1: Retrieve what's currently stored for key k_t
        retrieved = S @ K[t]              # (d_v,)

        # Step 2: Compute delta = new_value - old_value
        delta = V[t] - retrieved            # (d_v,)

        # Step 3: Update memory (erase-then-write)
        S = S + beta[t] * torch.outer(delta, K[t])  # (d_v, d)

        # Step 4: Read with query
        y_t = S @ Q[t]                    # (d_v,)
        outputs.append(y_t)

    return torch.stack(outputs)        # (T, d_v)
Interpretation: $\beta_t$ is a "learning rate" for the memory. With $\beta = 1$, the memory perfectly overwrites old values. With $\beta = 0$, no update happens. The model learns when to update aggressively vs. conservatively.

5. Why the Delta Rule Works: Mathematical Intuition

Consider the case where keys are orthonormal ($k_i^\top k_j = \delta_{ij}$) and $\beta = 1$:

$$S_t k_t = (S_{t-1} + (v_t - S_{t-1}k_t) k_t^\top) k_t = S_{t-1}k_t + v_t - S_{t-1}k_t = v_t$$

After the update, retrieving with the same key gives exactly the new value $v_t$, with the old value completely erased. For non-orthogonal keys, the update is approximate but still much better than pure accumulation.

Connection to Linear Regression $$\text{Delta rule} \iff \text{Online gradient descent on } \min_S \sum_t \|v_t - S k_t\|^2$$

The delta update $S_t = S_{t-1} + \beta_t (v_t - S_{t-1}k_t) k_t^\top$ is exactly the gradient step for minimizing the squared error between the stored value $Sk_t$ and the target $v_t$.
