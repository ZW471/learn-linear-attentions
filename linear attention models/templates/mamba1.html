Mamba-1: Selective State Spaces

Linear-Time Sequence Modeling with Selective State Spaces
Gu, Dao · 2023 · arXiv:2312.00752

1. The Problem with S4: No Selectivity

S4's parameters ($A$, $B$, $C$) are fixed after training — they don't change based on the input. This means S4 treats every token the same way, regardless of content.

The limitation: Consider processing "The cat sat on the mat [IMPORTANT] The key fact is...". S4 cannot selectively attend more to tokens after [IMPORTANT] because its dynamics are input-independent.

Mamba's solution: make $B$, $C$, and $\Delta$ functions of the input.

S4 (Time-Invariant)

$$x_k = \bar{A}x_{k-1} + \bar{B}u_k$$ $$\bar{A}, \bar{B} \text{ are the same at every step}$$

Mamba (Selective / Time-Varying)

$$x_k = \bar{A}_k x_{k-1} + \bar{B}_k u_k$$ $$\bar{A}_k, \bar{B}_k \text{ depend on } u_k$$

2. The Selection Mechanism

Mamba generates $B_t$, $C_t$, and $\Delta_t$ as linear projections of the input:

Input-Dependent Parameters $$B_t = s_B(x_t) = \text{Linear}_N(x_t) \quad \in \mathbb{R}^N$$ $$C_t = s_C(x_t) = \text{Linear}_N(x_t) \quad \in \mathbb{R}^N$$ $$\Delta_t = \text{softplus}(s_\Delta(x_t)) = \text{softplus}(\text{Linear}_1(x_t)) \quad \in \mathbb{R}^+$$

Then discretization happens per timestep using Zero-Order Hold (ZOH):

Per-Timestep Discretization (ZOH) $$\bar{A}_t = \exp(\Delta_t \cdot A) \qquad \bar{B}_t \approx \Delta_t \cdot B_t$$
Intuition for $\Delta$: The step size $\Delta_t$ controls how much the model "pays attention" to token $t$. Large $\Delta_t$ → the state changes a lot (remember this token). Small $\Delta_t$ → the state barely changes (ignore this token). $\Delta$ is the selection mechanism.

Tensor Shapes

u $(B, L, D)$ — input (batch, length, model dim)
A $(D, N)$ — diagonal state matrix (shared, learned)
B_t $(B, L, N)$ — input-dependent input projection
C_t $(B, L, N)$ — input-dependent output projection
$\Delta_t$ $(B, L, D)$ — input-dependent step sizes
x $(B, D, N)$ — hidden state

3. The Selective Scan Algorithm

Because $\bar{A}_t$ and $\bar{B}_t$ now change at every step, we can no longer use convolutions. Instead, Mamba uses a parallel scan.

Selective Scan (per feature d) $$x_t = \bar{A}_t \odot x_{t-1} + \bar{B}_t \odot u_t \qquad \in \mathbb{R}^{(D, N)}$$ $$y_t = \sum_n x_{t,d,n} \cdot C_{t,n} \qquad \in \mathbb{R}^{D}$$
PyTorch-like 
# Selective scan (simplified, single batch)
def selective_scan(u, A, B, C, delta):
    # u: (L, D), A: (D, N), B: (L, N), C: (L, N), delta: (L, D)
    L, D = u.shape
    N = A.shape[1]
    x = torch.zeros(D, N)           # hidden state (D, N)
    outputs = []

    for t in range(L):
        # Step 1: Discretize (per-timestep!)
        A_bar = torch.exp(delta[t, :, None] * A)  # (D, N)
        B_bar = delta[t, :, None] * B[t, None, :]   # (D, N)

        # Step 2: State update
        x = A_bar * x + B_bar * u[t, :, None]       # (D, N)

        # Step 3: Output
        y_t = (x * C[t, None, :]).sum(dim=-1)       # (D,)
        outputs.append(y_t)

    return torch.stack(outputs)      # (L, D)

4. Hardware-Aware Implementation

The naive scan above would materialize the full state tensor $(B, L, D, N)$ in GPU HBM (slow memory). Mamba avoids this with kernel fusion:

GPU Memory Hierarchy

SRAM (on-chip): ~20 MB, ~19 TB/s
Fused kernel keeps state here
HBM (off-chip): ~80 GB, ~2 TB/s
Only reads input, writes output

Fused Operations

Three operations fused into one CUDA kernel:

  1. Discretization: $\Delta_t, A \rightarrow \bar{A}_t, \bar{B}_t$
  2. Selective scan: $x_t = \bar{A}_t x_{t-1} + \bar{B}_t u_t$
  3. Output: $y_t = C_t^\top x_t$

The state $(D, N)$ stays in SRAM the entire time — never written to HBM.

5. The Mamba Block Architecture

One Mamba block (replacing a Transformer block):

PyTorch-like 
class MambaBlock(nn.Module):
    def __init__(self, D, N=16, expand=2):
        E = D * expand
        self.in_proj  = nn.Linear(D, 2 * E)     # (D) -> (2E) split into x, z
        self.conv1d   = nn.Conv1d(E, E, 4, groups=E, padding=3)
        self.ssm_proj = nn.Linear(E, N+N+1)     # -> B(N), C(N), delta(1)
        self.A_log    = nn.Parameter(...)          # (E, N) learned
        self.out_proj = nn.Linear(E, D)

    def forward(self, u):                         # u: (B, L, D)
        xz = self.in_proj(u)                       # (B, L, 2E)
        x, z = xz.chunk(2, dim=-1)                 # each (B, L, E)
        x = self.conv1d(x.transpose(1,2)).transpose(1,2)
        x = F.silu(x)                              # (B, L, E)

        # Input-dependent SSM parameters
        ssm_params = self.ssm_proj(x)              # (B, L, 2N+1)
        B, C, dt = split(ssm_params)               # B,C:(B,L,N) dt:(B,L,1)
        A = -torch.exp(self.A_log)                 # (E, N)

        y = selective_scan(x, A, B, C, dt)          # (B, L, E)
        y = y * F.silu(z)                            # gating
        return self.out_proj(y)                     # (B, L, D)
