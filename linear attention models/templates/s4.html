S4: Structured State Spaces

Efficiently Modeling Long Sequences with Structured State Spaces
Gu, Goel, Re · ICLR 2022 · arXiv:2111.00396

1. The Big Idea

S4 models sequences by mapping them through a continuous-time state space model, then discretizing it for computation. The key breakthrough is making this efficient using the HiPPO matrix and structured matrix decompositions.

If you know Transformers: Think of S4 as replacing the $T \times T$ attention matrix with a fixed-size $N$-dimensional hidden state that compresses the entire history. During training, this state is computed as a convolution (parallelizable). During inference, it's a recurrence (constant time per token).

Data Flow Overview

2. Continuous-Time State Space Model

The starting point is a continuous-time linear system with state $x(t) \in \mathbb{R}^N$:

Continuous SSM $$x'(t) = Ax(t) + Bu(t)$$ $$y(t) = Cx(t) + Du(t)$$
A $\in \mathbb{R}^{N \times N}$ — state transition matrix (how state evolves)
B $\in \mathbb{R}^{N \times 1}$ — input projection (how input enters state)
C $\in \mathbb{R}^{1 \times N}$ — output projection (how to read from state)
D $\in \mathbb{R}$ — skip connection (direct input-to-output)
PyTorch-like 
# Continuous-time SSM (conceptual)
class ContinuousSSM:
    def __init__(self, N):
        self.A = nn.Parameter(torch.randn(N, N))  # (N, N)
        self.B = nn.Parameter(torch.randn(N, 1))  # (N, 1)
        self.C = nn.Parameter(torch.randn(1, N))  # (1, N)
        self.D = nn.Parameter(torch.zeros(1))     # (1,)

    # x'(t) = A @ x(t) + B * u(t)
    # y(t)  = C @ x(t) + D * u(t)
Analogy: In a Transformer, the "state" is the full sequence of past key-value pairs ($O(T)$ memory). In an SSM, the state is a fixed $N$-dimensional vector ($O(N)$ memory, typically $N = 64$), no matter how long the sequence.

3. The HiPPO Matrix

The matrix $A$ is initialized with the HiPPO-LegS (High-Order Polynomial Projection Operator, Legendre Shifted) matrix. This is the key to S4's ability to model long-range dependencies.

HiPPO-LegS $$A_{nk} = -\begin{cases} \sqrt{2n+1}\sqrt{2k+1} & \text{if } n > k \\ n+1 & \text{if } n = k \\ 0 & \text{if } n < k \end{cases}$$ $$B_n = \sqrt{2n+1}$$

The HiPPO matrix makes the hidden state $x(t)$ track the coefficients of a Legendre polynomial approximation of the input history. Each state dimension remembers a different polynomial basis function.

HiPPO Matrix Visualization (N=4)

Why Legendre polynomials? They form an orthonormal basis that can approximate any function over $[0, t]$. As $N$ increases, the approximation improves. With $N = 64$, the state can accurately recall sequences thousands of steps long.

4. Discretization: From Continuous to Discrete

Real sequences are discrete (token-by-token), so we discretize with step size $\Delta > 0$ using the bilinear method (Tustin's method):

Bilinear Discretization $$\bar{A} = \left(I - \frac{\Delta}{2}A\right)^{-1}\left(I + \frac{\Delta}{2}A\right)$$ $$\bar{B} = \left(I - \frac{\Delta}{2}A\right)^{-1} \Delta B$$ $$\bar{C} = C$$
PyTorch-like 
# Bilinear discretization
def discretize_bilinear(A, B, C, delta):
    # A: (N, N), B: (N, 1), delta: scalar
    I = torch.eye(A.shape[0])
    inv = torch.linalg.inv(I - (delta / 2) * A)  # (N, N)
    A_bar = inv @ (I + (delta / 2) * A)               # (N, N)
    B_bar = inv @ (delta * B)                          # (N, 1)
    return A_bar, B_bar, C

$\Delta$ controls the "resolution" — how much continuous time one discrete step corresponds to. Small $\Delta$ = fine-grained, large $\Delta$ = coarse.

5. The Duality: Recurrence vs Convolution

Process tokens one at a time, maintaining state. Best for inference.

Recurrence $$x_k = \bar{A}\,x_{k-1} + \bar{B}\,u_k \qquad y_k = \bar{C}\,x_k$$
PyTorch-like 
# Recurrent mode: O(L*N^2) sequential, O(N) memory
def ssm_recurrent(A_bar, B_bar, C_bar, u):
    # u: (L,) input sequence
    # x: (N,) hidden state
    x = torch.zeros(N)             # (N,)
    outputs = []
    for k in range(L):
        x = A_bar @ x + B_bar.squeeze() * u[k]  # (N,)
        y_k = C_bar @ x                           # (1,)
        outputs.append(y_k)
    return torch.stack(outputs)     # (L,)

Step-by-Step Recurrence

Compute the entire output in parallel via convolution. Best for training.

Convolution Kernel $$\bar{K} = (\bar{C}\bar{B},\; \bar{C}\bar{A}\bar{B},\; \bar{C}\bar{A}^2\bar{B},\; \ldots,\; \bar{C}\bar{A}^{L-1}\bar{B}) \in \mathbb{R}^L$$ $$y = \bar{K} * u \quad \text{(computed via FFT)}$$
PyTorch-like 
# Convolution mode: O(L log L) parallel via FFT
def ssm_convolution(A_bar, B_bar, C_bar, u, L):
    # Build kernel: K[i] = C_bar @ A_bar^i @ B_bar
    K = torch.zeros(L)
    A_power = torch.eye(N)          # A^0 = I
    for i in range(L):
        K[i] = C_bar @ A_power @ B_bar  # scalar
        A_power = A_power @ A_bar       # (N, N)

    # Convolve via FFT: O(L log L)
    y = torch.fft.irfft(
        torch.fft.rfft(K, n=2*L) * torch.fft.rfft(u, n=2*L)
    )[:L]
    return y  # (L,)

6. Efficient Computation: NPLR & DPLR

Naively computing the convolution kernel $\bar{K}$ requires $L$ matrix powers of $\bar{A}$, which is $O(LN^2)$. S4 avoids this by exploiting the structure of the HiPPO matrix.

Normal Plus Low-Rank (NPLR)

NPLR Decomposition $$A = V\Lambda V^* - PQ^\top$$

Where $V \in \mathbb{C}^{N \times N}$ is unitary, $\Lambda$ is diagonal, and $P, Q \in \mathbb{R}^{N \times r}$ are low-rank factors (typically $r = 1$ or $2$).

Diagonal Plus Low-Rank (DPLR)

DPLR (in conjugated basis) $$A = \Lambda - PQ^* \quad \text{where } \Lambda \text{ is diagonal}$$

In DPLR form, the generating function for the kernel reduces to a Cauchy kernel:

Cauchy Kernel $$\hat{K}(\omega) \propto \sum_i \frac{\tilde{C}_i \tilde{B}_i}{\omega - \Lambda_i}$$

This can be computed in $\tilde{O}(N + L)$ time using fast Cauchy matrix algorithms, making S4 efficient for very long sequences.

The efficiency pipeline: HiPPO matrix → NPLR decomposition → conjugate to DPLR → kernel via Cauchy product → output via FFT. Total: $O((N + L) \log^2(N + L))$.
