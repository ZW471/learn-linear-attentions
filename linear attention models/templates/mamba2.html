Mamba-2: State Space Duality

Transformers are SSMs: Generalized Models and Efficient Algorithms Through Structured State Space Duality
Dao, Gu · 2024 · arXiv:2405.21060

1. The Key Insight: SSMs = Structured Masked Attention

Mamba-2 reveals that SSMs and attention are not different models — they're two views of the same matrix computation.

Linear (Recurrent) Form

$$h_t = a_t \cdot h_{t-1} + B_t \cdot x_t^\top$$ $$y_t = C_t^\top \cdot h_t$$

Process one token at a time. State $h_t \in \mathbb{R}^{N \times P}$.
Complexity: $O(T \cdot N \cdot P)$

Quadratic (Attention) Form

$$Y = \underbrace{(L \circ CB^\top)}_{M} \cdot X$$

Compute full $T \times T$ matrix, then multiply. Like attention!
Complexity: $O(T^2)$

The duality: Both forms compute the exact same output $Y$. The linear form is better for long sequences; the quadratic form is better for short sequences (it uses fast matrix multiplications). Mamba-2 combines them.

2. The Semiseparable Matrix $M$

When we unroll the SSM recurrence, we get a structured matrix $M$ where:

Semiseparable Matrix $$M_{ij} = C_i^\top \cdot \left(\prod_{k=j+1}^{i} A_k\right) \cdot B_j \quad \text{for } i \geq j, \quad M_{ij} = 0 \text{ for } i < j$$

Critical restriction in Mamba-2: $A_t$ is a scalar (not a matrix). This allows factoring:

Factored Form (scalar A) $$M = L \circ (C \cdot B^\top)$$

Where $L$ is the 1-semiseparable mask (the "discount factor" matrix):

1-SS Mask $$L_{ij} = \prod_{k=j+1}^{i} a_k \quad (i \geq j), \qquad L_{ij} = 0 \quad (i < j)$$

Visualizing the Factorization: $M = L \circ (CB^\top)$

Compare to standard attention: In a Transformer, $M = \text{softmax}(QK^\top / \sqrt{d})$. In Mamba-2, $M = L \circ (CB^\top)$. The structure of $L$ (exponential decay) replaces the softmax, and $C, B$ play the roles of $Q, K$.

3. Three Ways to Compute the Same Thing

$$G = C \cdot B^\top \in \mathbb{R}^{T \times T} \quad \text{(like } QK^\top \text{)}$$ $$M = L \circ G \quad \text{(apply decay mask)}$$ $$Y = M \cdot X \quad \text{(like attention output)}$$
PyTorch-like 
def ssd_quadratic(A, B, C, X):
    # A: (T,) scalar, B: (T,N), C: (T,N), X: (T,P)
    L = build_1ss_mask(A)         # (T, T) decay mask
    G = C @ B.T                    # (T, T) like QK^T
    M = L * G                      # (T, T) element-wise
    Y = M @ X                      # (T, P) output
    return Y

Complexity: $O(T^2 N + T^2 P)$. Good when $T$ is small.

$$h_t = a_t \cdot h_{t-1} + B_t \otimes X_t \quad \in \mathbb{R}^{N \times P}$$ $$y_t = C_t^\top \cdot h_t \quad \in \mathbb{R}^P$$
PyTorch-like 
def ssd_recurrent(A, B, C, X):
    # A: (T,) scalar, B: (T,N), C: (T,N), X: (T,P)
    h = torch.zeros(N, P)          # state: (N, P)
    outputs = []
    for t in range(T):
        h = A[t] * h + torch.outer(B[t], X[t])  # (N,P)
        y_t = C[t] @ h                            # (P,)
        outputs.append(y_t)
    return torch.stack(outputs)    # (T, P)

Complexity: $O(T \cdot N \cdot P)$. Good when $T$ is large.

Mamba-2's actual algorithm: split the sequence into chunks of size $Q$, use quadratic form within chunks and recurrent form between chunks.

SSD Chunked Algorithm $$\text{Step 1 (intra-chunk):} \quad Y_{\text{diag}} = (L_c \circ C_c B_c^\top) \cdot X_c \quad \text{(matmul, } O(Q^2) \text{)}$$ $$\text{Step 2 (chunk states):} \quad h_c = \sum_{s} \bar{A}_{c,s} \cdot B_s \otimes X_s$$ $$\text{Step 3 (inter-chunk):} \quad h_c \leftarrow \bar{a}_c \cdot h_{c-1} + h_c \quad \text{(scan on } T/Q \text{ chunks)}$$ $$\text{Step 4 (state → output):} \quad Y_{\text{off}} = C_c^\top \cdot h_{c-1} \cdot \text{decay}$$
PyTorch-like 
def ssd_chunked(A, B, C, X, Q=64):
    # Split into chunks of size Q
    num_chunks = T // Q
    # Step 1: Intra-chunk (quadratic, uses tensor cores)
    for c in range(num_chunks):
        L_c = build_1ss_mask(A[c*Q:(c+1)*Q])          # (Q, Q)
        M_c = L_c * (C_c @ B_c.T)                       # (Q, Q) matmul!
        Y_intra[c] = M_c @ X_c                          # (Q, P) matmul!

    # Step 2-3: Inter-chunk state passing (parallel scan)
    states = parallel_scan(chunk_states, decay_per_chunk)

    # Step 4: Add inter-chunk contribution
    Y = Y_intra + state_contribution
    return Y

Complexity: $O(TN + T^2/Q)$. Steps 1,2,4 are matmul-dominated (use tensor cores, up to 16x faster than non-matmul FLOPs). Step 3 is a scan on only $T/Q$ elements.
